## 一、产品定位

**产品名称（暂定）**: KS Agent
**一句话定位**: "用自然语言，5 秒生成可部署的 Kickstart 文件"

**与 OpenShadow 关系**: KS Agent 是 OpenShadow 的商业插件，专注于 Linux 批量装机场景的 AI 能力输出。

---

## 二、核心价值主张

| 用户 | 痛点 | 解决方案 |
|------|------|---------|
| 普通运维 | 不会写 KS 语法 | 对话式生成，0 学习成本 |
| 专家运维 | 反复改 KS，效率低 | 自然语言微调，秒级生成 |
| 销售/售前 | 产品缺 AI 亮点 | 高感知度 AI 能力加持 |

---

## 三、目标用户

- **内部**: 浪潮运维团队
- **外部**: 企业客户运维团队
- **场景**: 以浪潮服务器为主，混合厂商客户设备

---

## 四、第一阶段产品边界

### 做（MVP）

1. **对话式 KS 生成**
   - 输入: 自然语言描述（"一台 4 核 16G 的 Web 服务器，IP 192.168.1.100"）
   - 输出: 完整可用的 Kickstart 文件内容
   - 支持 Linux (CentOS/RHEL/Ubuntu)
   - LLM 调用: MiniMax / DeepSeek / Qwen / GLM（OpenShadow 现有 provider）

2. **生成结果预览**
   - 在 OpenShadow Chat UI 中展示 KS 文件内容
   - 支持复制 / 下载为 .ks 文件

3. **API 上传至浪潮 OS 部署产品**
   - 对接浪潮 OS 部署产品的 REST API
   - 实现 KS 文件上传接口调用
   - 前提: 获取 OS 部署产品 API 文档

4. **Windows KS 探索（不承诺）**
   - Windows 采用 Autounattend.xml，语法与 KS 不同
   - 第一阶段不实现，第二阶段评估

### 不做（第二阶段）

- 部署状态监控 / 巡逻推荐
- 模板版本管理 / KS 文件库
- 多机器批量生成
- 部署前校验（IP 冲突、磁盘空间预检等）

## 五、技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                     OpenShadow Desktop                       │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              KS Agent Plugin (OpenShadow)           │    │
│  │  1. 用户输入自然语言需求                             │    │
│  │  2. 组装 prompt → 调用 LLM                         │    │
│  │  3. 解析 KS 文件内容                                │    │
│  │  4. 预览 / 下载 KS                                  │    │
│  │  5. 调用浪潮 OS 部署产品 API（上传 KS）            │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ REST API
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              浪潮 OS 部署产品（独立产品）                    │
│  - KS 文件存储                                               │
│  - PX 菜单配置                                               │
│  - 部署任务触发                                               │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ PXE / DHCP
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      目标机器（待装机）                      │
│ 浪潮服务器 + 混合厂商服务器                                   │
└─────────────────────────────────────────────────────────────┘
```

### 关键模块

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| LLM 调用 | OpenShadow 现有 pi-sdk | MiniMax / DeepSeek / Qwen / GLM |
| KS 生成 Prompt | 自定义 | 需要构建 KS 领域知识 prompt |
| API 对接 | REST / axios | 对接浪潮 OS 部署产品上传接口 |
| 插件注册 | OpenShadow Plugin System | 作为 builtin 插件集成 |

### 依赖项（需获取）

- 浪潮 OS 部署产品 API 文档（KS 文件上传接口）
- OS 部署产品的 baseUrl / auth 方式

## 八、风险与对策

| 风险 | 概率 | 对策 |
|------|------|------|
| LLM 生成 KS 语法错误 | 中 | 构建 few-shot prompt，提供正确 KS 示例 |
| 浪潮 OS 部署产品 API 不开放 | 低 | 有源码 + 团队支持，可内部协调 |
| Windows KS 复杂度超预期 | 高 | 第一阶段只做 Linux，Windows 降级 |
