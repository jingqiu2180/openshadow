OpenShadow 使用手册 v0.3.45+

一份写给最终用户的实用指南 —— 不需要会编程也能看懂。基于当前已发布的 0.3.45 版本编写，覆盖全部 16 个设置标签与频道协作系统。

一句话定位：OpenShadow 是一个常驻你电脑的 AI 助手。它既能像聊天机器人一样对话、读文件、改代码、跑命令、上网，也能在频道里和多人协作、在你离开时按你贴的"笺"指令默默干活。

1. 这是什么

OpenShadow 是一个"桌面版 AI 工作搭档"。和普通网页聊天机器人的区别：

  • 能真正操作你的电脑：读/写文件、执行命令、搜索代码、列目录 —— 都在你授权的工作区内。
  • 能上网、能生成图片/视频：配置好相应服务后，助手可直接浏览网页、产出多模态内容。
  • 频道协作：内置类似群聊的空间，你可建频道、拉成员，AI 助手作为参与者加入一起讨论。
  • 后台自动工作：贴一张"笺(jian)"便签，关掉窗口它也会继续做，做完汇报。
  • 多助手、多模型：可同时养多个性格不同的 Agent，各自有记忆、各自对接不同大模型。

2. 安装与首次启动

安装

下载 OpenShadow-*.Windows-x64.exe 双击安装，默认装到：

C:\Users\你的用户名\AppData\Local\Programs\OpenShadow

安装过程可能"看起来卡住"：Windows Defender 会对 500MB 左右的安装包逐文件扫描，可能要等 15–20 分钟。只要任务管理器里安装进程还在、C 盘没报错，就耐心等，不要反复点。

首次启动向导

第一次打开会弹出设置向导，引导你完成：欢迎 → 选择供应商/填 API Key → 选择模型 → 选择工作台目录 → 完成。

  • "下一步"按钮在 0.3.44 之后全程可点、始终推进；没填完的字段会留到「设置」里补，不会卡死。
  • 向导里选的工作台目录从 0.3.45 起会真正生效（此前版本会回退成默认目录，是个 bug，已修）。
想重新跑一次向导？ 关闭所有 OpenShadow 窗口（含右下角托盘图标），打开配置文件 %USERPROFILE%\.openshadow\config.json，把 "completed": true 改成 false，再启动即可。

3. 主界面导览

主窗口大致分几块：

区域作用
左侧边栏会话列表（历史对话）、频道列表（DM/群聊，见第 6 节）、新建对话、桥接、动态（activity）。可折叠。
中间聊天区和当前助手对话。底部有模型选择下拉框和输入框。
顶部标签栏在「对话 / 频道 / 插件」等视图间切换。
右上角 ⚙ 按钮真正的设置入口 → 打开设置弹窗（见第 10 节）。
顶部"书桌"开关打开笺（jian）侧栏，写异步指令便签（见第 7 节）。
注意：不要用右下角托盘图标的"设置"。 在旧版本里，托盘的"设置"窗口是空白的（已知 bug）。所有设置请从主聊天窗右上角的 ⚙ 齿轮进入。

4. 配置供应商与模型

这是最容易卡住新人的地方，重点讲。

下拉框的数据从哪来？

聊天窗口右下角的"模型选择"下拉框，完全来自你在设置里配置的供应商。没有配置供应商 → 下拉框显示"未配置模型"，为空。

配置链路：

  1. 打开 ⚙ 设置 → 供应商（Providers） 标签。
  2. 从内置预设里挑一个（如 MiniMax、DeepSeek、硅基流动…），填入你自己的 API Key 和要用的模型。
  3. 保存后，模型才会出现在聊天下拉框里。

内置 32 个供应商预设（0.3.44 起）

软件自带 32 个常见供应商模板（MiniMax、DeepSeek、Qwen、智谱 GLM、Kimi、百度千帆、腾讯混元、火山方舟、硅基流动、Groq、Mistral、StepFun 等）。预设只给模板，不含 Key —— 因为软件是分发给团队、每人填自己的 Key 用的。

供应商标签里还能配什么？

  • API Key / 令牌：各供应商的密钥。
  • 模型：主模型 / 小模型 / 大模型（main/small/large）分别指定，可设 Base URL、上下文长度、思考强度（reasoning effort）。
  • 联网搜索：接入 Tavily / Brave / Serper / AnySearch 等搜索服务，让助手能联网查资料。

显示名是怎么来的？

你在配置里写的是模型 ID（如 MiniMax-M3），界面会翻译成易读名 "MiniMax M3"。这很正常。

快捷改法（高级）： 也可直接编辑配置文件 %USERPROFILE%\.openshadow\config.json 的 providers 数组，把 models.main 改成 供应商id::模型名，保存后重启生效。

5. 助手（Agent）与我

OpenShadow 支持多个助手并行，每个有独立记忆、性格和工作台。

  • 打开 ⚙ 设置 → 助手（Agent） 标签：配置助手的人设、角色、头像、记忆（"源/yuan"）。可切换 / 新建 / 编辑助手。
  • 内置人格预设有 Hanako、Butter、Ming、Kong 等（各有不同风格），创建时可选用。
  • 我（Me） 标签：填写你自己的个人资料 / 自我介绍，让助手更了解你，回答更贴合。

侧边栏切换会话时，实际上也是在切换"当前由哪个助手服务"。

6. 频道（Channels）Beta

频道是一个内置的群聊 / 协作空间，类似 Slack、Discord 的频道机制，而且 AI 助手会作为参与者加入。

在哪里？

  • 左侧边栏有一个频道区块（标注 Beta），分 DM（私聊） 和 Group（群聊） 两个列表。
  • 顶部标签栏可切到「频道」视图。
  • 首次使用需开启频道开关（会弹一次确认提示），开启后加载频道列表。

怎么用？

  1. 点频道区块的「＋ 新建」按钮 → 弹出创建频道窗口 → 填名称创建群聊频道。
  2. 在频道里：有消息流、成员列表、输入框，和群聊一样用。
  3. AI 助手是频道成员：右侧栏有「Agent 设置」和「Agent 动态」两个面板，可配置它在该频道的角色与查看它的活动。
  4. 可发起 DM 私聊（一对一频道）。
  5. 频道有未读红点提示，部分频道可设为只读。
用途示例：把助手拉进"本周需求评审"群，大家在频道里@它查代码、写方案；或一个人开多个频道按主题分工作业，每个频道配不同的助手人格。

7. 笺（jian）与自动巡检

这是 OpenShadow 区别于普通聊天机器人的灵魂功能。

什么是"笺"？

"笺"(jiān) 是一张指令便签：你在任意文件夹里放一个 jian.md 文件，写下要它做的事。助手在后台自动巡检时会扫到这张笺并读取执行，不必一直开着聊天窗口。可以理解为：你到处贴便利贴，助手按路线巡店时逐张处理。

怎么用？

  1. 点主窗口顶部的 "书桌" 开关（侧边栏标为"书桌"，打开的是笺面板），展开笺侧栏。
  2. 在任意工作台目录下新建一个 jian.md 文件，里面写指令（例如"整理这个文件夹里的图片，按日期归类"）。
  3. 助手按心跳/定时（cron）扫描工作区，读到 jian.md 就执行，并实时更新任务状态。
适合场景：你下班前贴一张笺"明天早上把本周日志汇总成一份报告"，第二天回来报告已经在桌上了。你甚至不需要打开聊天窗。巡检的开关与频率在 ⚙ 设置 → 工作台 → 巡检 里配置。

8. 工作台与文件

每个助手有一个"工作台（Desk）"工作目录，是它读写的主战场。

  • 可在 ⚙ 设置 → 工作台（Work）→ 工作台目录 查看 / 切换当前工作台目录。
  • 工作台里可以拖文件、预览资源、贴笺（jian.md），助手在巡检时会读取这些素材。
  • 支持注入 AGENTS.md / CLAUDE.md 之类的"项目说明"文件，让助手自动了解该目录的规范。
  • 0.3.45 修复：向导里选的工作台目录现在会真正设为激活工作台，不再是默认的 Desktop\OH-WorkSpace。

9. 会话管理

  • 左侧边栏列出所有对话历史，可搜索、归档、恢复。
  • 对话里可以引用(quote)某条消息，把上下文带入后续讨论。
  • 不同助手的会话互相独立，记忆不串。

10. 设置中心详解（16 个标签）

点右上角 ⚙ 打开设置弹窗，左侧是一列标签。下面逐个说明它们干什么。搜索框支持中英文关键词（如"代理""theme"）快速定位。

助手（Agent）/ 我（Me）

  • 助手：配置 AI 的人设、角色、头像、记忆（源）。新建/切换助手都在这里。
  • 我：填写你自己的资料/自我介绍，让助手更懂你。

界面（Interface）

  • 主题：浅色 / 深色 / 纸张纹理（paper texture）。
  • 字体：正文字体、Markdown 字体（宋体/黑体等）。
  • 语言与时区：界面语言、时区、地区。
  • 快捷键：查看/自定义键盘快捷键，含语音快捷键。

通用（General）

  • 启动：开机自启、保持后台运行、锁屏行为。
  • 快速聊天：小窗快速唤起、是否保留输入框内容。
  • 通知：轮次完成提醒等。

浏览器（Browser）

  • Cookie / 站点数据：管理并清除助手上网用的 Cookie。
  • 助手上网行为：控制助手打开网页时用新标签页还是当前标签页等。

工作台（Work）

  • 工作台目录：设置默认工作目录，并可注入 AGENTS.md/CLAUDE.md 项目说明。
  • 巡检（心跳）：配置后台自动任务的开关、频率与权限 —— 也就是第 7 节"笺"自动执行的引擎。

技能（Skills）

安装与管理技能包（可复用的能力模块，如特定流程、工具）。让助手掌握新本领。

桥接（Bridge）

把 OpenShadow 接到外部社交平台（微信、Telegram 等），让你在手机/别处也能通过聊天驱动助手。

媒体（Media）

  • 图片生成、视频生成：接入对应服务后，助手可产出图像/视频。
  • 语音识别 / 转录：语音转文字能力配置。

共享（Sharing）

对话的分享设置：截图、生成分享卡片、水印等。

访问（Access）

让你从手机 / 局域网 / 远程访问本机助手：PWA、端口、二维码等。方便手机端随时用。

插件（Plugins）

插件市场与已装插件管理，含开发工具与"完全访问"开关（给插件更高权限）。

实验（Experiments）

Beta / 预览功能开关，例如 电脑使用（Computer Use） 等前沿能力。按需开启。

安全（Security）

  • 沙盒：联网沙盒、文件备份、归档对话管理。
  • 网络代理：HTTP/SOCKS 代理或直连设置（访问境外模型时常需要）。

关于（About）

查看版本号、检查更新、许可证信息。

11. 安全与沙箱

OpenShadow 有双层沙箱：应用层路径守卫 + 系统级隔离。默认只允许访问你显式授权的工作区目录。

  • 设置 → 安全（Security） 里可管理"允许访问的目录白名单（workspaceRoots）"。
  • 向导里选的工作台目录会自动加入白名单。
  • 助手越权访问白名单外的路径会被拒绝 —— 这是保护，不是 bug。

12. 托盘与快捷键

  • 窗口关闭后软件常驻右下角托盘（单实例，重复双击不会开多个）。
  • 托盘图标右键可退出；但如前所述，托盘的"设置"窗口在旧版本是空白的，设置请走 ⚙。
  • 常见快捷键：Ctrl/Cmd + K 类全局操作、Esc 关闭弹窗（具体以界面提示为准）。

14. 配置与排障

配置文件位置

C:\Users\你的用户名\.openshadow\config.json

所有设置（供应商、模型、工作台、助手、向导完成标志、频道开关等）都在这。重装软件不会删除它，所以你的配置和 Key 都保得住。

常用排障

  • 软件没反应 / 卡住：先彻底退出（包括托盘图标右键退出），再重开。Electron 是单实例，老进程可能还占着。
  • 想重置向导：把 wizard.completed 改为 false 后重启。
  • 升级卸载框死循环：结束安装进程，删除注册表 HKCU\Software\Microsoft\Windows\CurrentVersion\Uninstall 下 OpenShadow 的旧卸载项，再重装即判定为全新安装。
  • 日志：异常时可看安装目录下的 logs/ 或系统 %TEMP% 相关日志辅助定位。
