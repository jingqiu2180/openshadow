Single-process orchestrator (`hub/index.ts` `Hub` class) that owns one `EventBus`, a `ChannelRouter`, `GuestHandler`, `Scheduler`, `DmRouter`, and an `AgentPhoneActivityStore`, then wires them back into the injected `HanaEngine` via `engine.setHubCallbacks` / `engine.setEventBus`. Dependency direction is strictly Hub → Engine; Engine never imports Hub.

- `event-bus.ts` `EventBus` — in-memory pub/sub with session-path-indexed subscribers plus a request/handle RPC pattern (first non-SKIP handler wins, with timeout). `event-bus-capabilities.ts` declares a frozen `BUILTIN_EVENT_BUS_CAPABILITIES` registry (session/agent/provider/media/task types) consumed by `EventBusCapabilityDirectory` so callers can introspect capabilities.
- `channel-router.ts` `ChannelRouter` — wraps `createChannelTicker` to poll `engine.channelsDir`, builds per-channel tool sets (`channel_read_context`, `channel_reply`, `channel_pass`) and delegates execution to `runAgentPhoneSession` from `agent-executor.ts`; also posts `channel_new_message` events on write.
- `dm-router.ts` `DmRouter` — 1v1 variant of the same flow: reads `agents/<id>/dm/<peer>.md`, runs up to `MAX_ROUNDS=3` phone turns, writes both sides' dm files, emits `dm_new_message`.
- `guest-handler.ts` `GuestHandler` — lightweight wrapper that prefixes guest text and injects a context tag before delegating to `engine.executeExternalMessage`.
- `scheduler.ts` `Scheduler` — owns per-agent `heartbeat` instances (via `lib/desk/heartbeat.js`) and a single Studio `cron-scheduler`; all background work goes through `engine.executeIsolated` with `activityType`/`persist` metadata and broadcasts `activity_update` on completion. Also hosts `FreshCompactMaintainer` for daily bridge memory compaction.
- `agent-executor.ts` — shared runtime used by Channel/Dm routers: `runAgentSession` (ephemeral temp session) and `runAgentPhoneSession` (persistent per-conversation session with projection/runtime sidecars, model override, scoped memory search, usage ledger recording, and abort registration).

Lifecycle: `Hub.initSchedulers()` starts heartbeat + channels; `pauseForAgentSwitch`/`resumeAfterAgentSwitch` coordinate engine restarts without breaking long-running channel ticks.
