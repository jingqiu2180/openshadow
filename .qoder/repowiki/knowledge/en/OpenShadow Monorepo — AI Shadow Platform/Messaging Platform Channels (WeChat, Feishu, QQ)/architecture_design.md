Each file (`wechat.ts`, `feishu.ts`, `qq.ts`) is an independent channel module following the same shape: it exports a typed `*Message` request interface, a `createXxxChannel(agent)` factory that builds a Hono app with a `/health` GET and a `/webhook` POST, and a `createXxxServer(agent, port)` helper returning `{ fetch, port, start }` for local testing. The only internal dependency is on `../core/agent.js`; there is no shared base class or registry — channels are composed at the application layer by wiring each factory into a parent server. Incoming payloads are parsed via `c.req.json()` cast to the per-channel interface, filtered by `schema`/`msgtype`, then routed through `agent.chat([{ role: 'user', content }, userId])`. Reply logic is currently partial: WeChat returns a JSON response in its native format, while Feishu/QQ log the result and return a generic `{ code: 0 }` acknowledgement (marked TODO).
