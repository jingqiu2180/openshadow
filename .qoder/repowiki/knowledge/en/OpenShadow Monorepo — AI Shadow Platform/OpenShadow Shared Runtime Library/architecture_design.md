Two-tier split: `shared/` holds pure, importable constants/schemas/utilities (config schema, model refs, error bus, workspace paths, media types, Hana root resolution) that both the Electron main process and Node worker processes can require without side effects; `lib/` builds on top of those with in-memory runtime state — ActivityHub (broadcasts agent activity to UI), TaskRegistry (plugin-safe task/schedule registry persisted via shared safe-fs), DeferredResult coordination, subagent/workflow stores, and file I/O helpers. Cross-child wiring is done through these shared modules as the single source of truth for config shape, model identity keys, error semantics, and workspace layout, so every child module depends only on `shared/` plus a small set of `lib/` primitives rather than on each other.
