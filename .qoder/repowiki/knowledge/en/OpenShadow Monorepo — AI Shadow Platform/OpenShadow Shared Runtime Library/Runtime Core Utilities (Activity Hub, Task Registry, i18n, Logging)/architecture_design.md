Flat collection of independent single-responsibility modules under `lib/` consumed by the host process:
- `activity-hub.ts`: in-memory `ActivityHub` class backed by a `Map<string, entry>`; normalizes entries via `normalizeEntry`, optionally persists workflow/subagent kinds through an injected `WorkflowActivityStore` (`_store.upsert`), and fans out to subscribers via `_cbs` plus an optional event bus (`bus.emit({type:'agent_activity', entry})`).
- `task-registry.ts`: `TaskRegistry` class owning three Maps (`_handlers`, `_tasks`, `_schedules`) plus Node `setTimeout` timers. Handlers are registered by `type` and must expose `{ run?, abort }`. Tasks persist to a JSON file via `atomicWriteSync` from `shared/safe-fs.js`; on load, active tasks are marked `recovering`. Schedules compute `nextRunAt` via `resolveNextRunAt` and arm timers capped at `MAX_TIMER_DELAY`.
- `i18n.ts`: module-scoped singleton loading locale JSON files from `desktop/src/locales/<key>.json` keyed by `resolveKey(locale)` (zh-CN→zh, zh-TW→zh-TW, ja, ko, en fallback); `t(path, vars)` does dot-path lookup with `{k}` placeholder substitution.
- `time-utils.ts`: exports `DAY_BOUNDARY_HOUR = 4` and `getLogicalDay` / `getLogicalDayForDate` so all date-bounded logic shares the same 4am day boundary.
- `debug-log.ts` + `log-redactor.ts`: `DebugLog` class writes timestamped `[HH:MM:SS.mmm] [LEVEL] [MODULE] msg` lines to `~/.hanako/logs/YYYY-MM-DD_HH-MM-SS.log`, deduplicates identical adjacent entries, caps each file at 5 MB, and cleans up older than 7 days; `createModuleLogger(module)` returns `{log/info/warn/error}` that writes both console and file. `log-redactor.ts` is a thin re-export of `../shared/log-redactor.js`.
- `theme-registry.cjs`: CommonJS stub exporting a minimal default theme set (`THEMES`, `AUTO_OPTION`, `getThemeIds`, `getTheme`, `isDark`) — loaded as a CJS module by the host.

Dependency direction is one-way outward: these modules import from `shared/*` (safe-fs, log-redactor, hana-root) but are not imported by each other except `debug-log.ts → log-redactor.ts` and `task-registry.ts → debug-log.ts`.
