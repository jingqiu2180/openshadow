Five independent leaf modules with no internal dependencies on each other:
- `log-redactor.ts` (ESM) and `log-redactor.cjs` (CommonJS) are a deliberately duplicated pair — the CJS file is kept so Electron main / server code can require it without a bundler, while the ESM file serves browser/renderer consumers. Both export `{ redactLogText, redactLogValue, redactLogLabel, formatLogArgs }` and share identical regex families (`SECRET_KEY_PATTERN`, `SENSITIVE_OBJECT_KEY_RE`, `API_KEY_VALUE_RE`, PII patterns for credit cards, CN IDs, SSNs, emails, long random tokens). The value walker uses a `WeakSet` to break cycles and caps recursion at depth 8.
- `secret-custody.ts` provides a separate, key-set-driven masking layer (`maskObjectSecrets`, `resolveSecretPatch`, `collectSecretPatchPaths`) that treats values equal to the sentinel `MASKED_SECRET` as "already masked" so patches can preserve existing secrets instead of overwriting them.
- `safe-fs.ts` wraps Node `fs` with try/catch + `errorBus.report(AppError(...))` fallbacks for `safeReadFile`/`safeReadJSON`/`safeReadYAML`, plus an atomic write helper (`atomicWriteSync` writes `.tmp` then `rename`) and an all-or-nothing directory copy (`safeCopyDir`) with `.bak_` rollback.
- `safe-parse.ts` offers `safeParseJSON` and `safeParseResponse` that swallow parse/fetch errors via `AppError('CONFIG_PARSE' | 'FETCH_SERVER_ERROR')` and return a caller-supplied fallback.

Dependency direction: this module depends outward on `./errors.js` (`AppError`) and `./error-bus.js` (`errorBus`) but nothing inside the scope depends on anything outside except those two shared primitives; there are no cross-imports between the five files.
